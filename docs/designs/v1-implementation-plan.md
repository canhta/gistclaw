## Additional In-Plan Fixes (all accepted, from CEO + Eng reviews)

- Named provider error handling steps (Task 4) — rate limit, context window exceeded, refusal, timeout, malformed response
- Shell exec injection test step (Task 6) — highest-risk tool, explicit injection test cases required
- WAL mode and connection settings (Task 2) — prevents SSE/write contention
- Telegram long polling specified in Task 11 — correct for local-first self-hosted tool
- Approval fingerprint formula locked: `sha256(tool_name + normalized_args_json + target_path)`

**From Eng review:**
- Runtime package split trigger rule added to Global Guardrails

- SSE fan-out: in-memory subscriber map `map[RunID][]chan ReplayEvent` in `sse.go`

- Proactive compaction at 75% context window threshold (Task 4, Step 3c)

- `outbound_intents` table added to Task 2 schema

- MockProvider required for all run-engine tests (Task 4)

## Pre-Implementation Open Questions

These should be resolved before the relevant task starts, not mid-implementation:

1. **Config default path** (before Task 1): `~/.config/gistclaw/config.yaml` (XDG, recommended) vs `./gistclaw.yaml`
2. **Streaming default** (before Task 4): use streaming for all interactive runs; non-streaming for short structured extraction only
3. **Repo signals for onboarding shortlist** (before Task 9b): git log last 30 days, open diffs, directory structure — define the exact scan signals
4. **SSE event schema** (before Task 8): define canonical event types and payload before client/server diverge (tracked in TODOS.md)
